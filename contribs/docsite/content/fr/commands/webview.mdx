# `webview`

Sert la base de données du graphe de connaissances dans une visualisation web
interactive — déplacer et zoomer, filtrer par type de nœud, rechercher des
symboles et inspecter les arêtes d'un nœud.

Source : [`src/commands/webview_command.ts`](https://github.com/jeromeetienne/ts_knowledge_graph/blob/main/src/commands/webview_command.ts) ·
interface : [`contribs/webview`](https://github.com/jeromeetienne/ts_knowledge_graph/tree/main/contribs/webview)

## Synopsis

```bash
npx ts-knowledge-graph webview [options]
```

## Arguments

Aucun.

## Options

| Option | Valeur par défaut | Description |
| --- | --- | --- |
| `-o, --output-folder <dir>` | `./.ts_knowledge_graph` | Dossier de sortie ; la base de données Kùzu est lue depuis `<dir>/graph.kuzu`. |
| `-p, --port <port>` | `4173` | Port HTTP sur lequel écouter. |
| `-s, --source <dir>` | `.` | Racine de projet de repli pour les liens profonds GitHub, utilisée uniquement lorsque le graphe n'a enregistré aucune provenance de source (voir [Liens GitHub](#liens-github)). |

## Ce qu'elle fait

1. Résout le chemin de la base de données et vérifie qu'elle existe. Sinon, elle
   affiche une erreur (`database not found … — run extract then load first`),
   définit le code de sortie `1` et se termine.
2. **Résout la source GitHub.** Elle préfère la provenance que
   [`extract`](/commands/extract) a enregistrée dans la base de données — le
   commit exact et le préfixe interne au dépôt qui ont été analysés — et se
   rabat sur une détection en direct depuis `--source`. Une source trouvée est
   injectée sous la forme `window.GRAPH_SOURCE`, de sorte que chaque nœud peut
   créer un lien profond vers sa déclaration sur GitHub à ce commit ; lorsque ni
   l'une ni l'autre n'est disponible, les chemins de fichiers s'affichent en
   texte brut.
3. **Lit l'intégralité du graphe une seule fois au démarrage.** Elle ouvre le
   magasin Kùzu et exécute deux requêtes Cypher — une pour tous les nœuds, une
   pour toutes les arêtes — puis sérialise le résultat dans un extrait
   JavaScript qui affecte `window.GRAPH_DATA`. Elle journalise le nombre de
   nœuds et d'arêtes chargés.
4. Démarre un serveur `http` Node simple :
   - `GET /js_autogenerated/graph_data.js` renvoie l'extrait du graphe en
     mémoire. C'est le script que la page charge au démarrage.
   - Tout autre chemin est servi comme fichier statique depuis le répertoire
     `contribs/webview/web` empaqueté (`/` correspond à `index.html`).
   - Le chemin du fichier statique est normalisé et confiné à la racine web, de
     sorte que les requêtes ne peuvent pas s'échapper du répertoire par une
     traversée `..`. Tout ce qui se trouve en dehors, ou tout fichier manquant,
     renvoie `404 not found`.
5. Affiche l'URL et attend. Arrêtez-le avec `Ctrl+C`.

Les types MIME servis couvrent `.html`, `.css`, `.js`, `.json`, `.png` et
`.svg` ; tout le reste se rabat sur `application/octet-stream`.

## La visualisation

La page est un graphe interactif unique (Cytoscape) doté d'une barre latérale de
contrôle :

- **Rechercher** — trouver n'importe quel symbole ou fichier par son nom et y
  accéder.
- **Colorer par** — basculer l'encodage des nœuds entre **structurel** (type de
  nœud), **d'exécution** (une carte de chaleur du temps propre, après
  [`enrich`](/commands/enrich)) et **communauté** (l'index de module issu de
  [`cluster`](/commands/cluster), dessiné avec une légende).
- **Filtrer** — activer et désactiver les types de nœuds et les types d'arêtes
  pour isoler une couche (uniquement `CALLS`, par exemple, ou masquer les nœuds
  `Parameter`) ; masquer des communautés entières.
- **Inspecter** — cliquer sur un nœud pour ses détails et ses arêtes à un saut,
  et le suivre jusqu'à sa ligne exacte sur GitHub (ci-dessous).
- **Thème** — sombre ou clair, suivant la préférence du système.

### Liens GitHub

Lorsque le graphe servi porte une provenance de source — enregistrée par
[`extract`](/commands/extract) au moment de la construction, ou détectée en
direct depuis `--source` —, chaque nœud crée un lien vers sa déclaration sur
GitHub au commit exact qui a été analysé. Construite en dehors d'un dépôt git
cloné, et sans repli `--source`, la visualisation affiche des chemins en texte
brut à la place.

## Sortie

```
loaded ~390 nodes, ~1.3k edges from /…/.ts_knowledge_graph/graph.kuzu
✓ serving the knowledge graph at http://localhost:4173/
  press Ctrl+C to stop
```

## Exemples

```bash
# serve the default database on the default port
npx ts-knowledge-graph webview

# choose a port
npx ts-knowledge-graph webview --port 8080

# point GitHub links at a specific checkout when the graph recorded no source
npx ts-knowledge-graph webview --source ../my-project
```

## Notes et mises en garde

- **Le graphe est lu une seule fois, au démarrage.** Les modifications apportées
  à la base de données après le démarrage du serveur ne sont pas répercutées —
  réexécutez [`extract`](/commands/extract) et [`load`](/commands/load), puis
  redémarrez `webview` pour voir les changements.
- L'intégralité du graphe est envoyée au navigateur en une seule charge utile.
  Cela convient aux graphes que cet outil produit ; les très grands graphes
  produiront un téléchargement initial volumineux.
- Garder la base de données ouverte ici peut bloquer un [`load`](/commands/load)
  concurrent. Arrêtez le serveur avant de recharger.
- Les ressources statiques sont résolues relativement au propre module de la
  commande, de sorte que la même commande fonctionne aussi bien depuis `src/`
  (via `tsx`) que depuis le paquet `dist/` publié.
