# Référence des commandes de la CLI

Cette section documente chaque commande enregistrée par la CLI
`ts-knowledge-graph` ([npx ts-knowledge-graph](https://www.npmjs.com/package/ts-knowledge-graph)).
Il y a une page par commande ; chacune explique en détail les arguments de la
commande, ses options, la requête de graphe sous-jacente, le format de sortie et
les mises en garde.

## Invocation

Le binaire publié est `ts-knowledge-graph`. Chaque exemple de cette documentation
l'exécute avec `npx` (aucune installation globale requise) :

```bash
npx ts-knowledge-graph <command> [arguments] [options]
```

## Le pipeline

Les commandes se répartissent en groupes qui s'exécutent à peu près dans l'ordre —
chaque étape produit un artefact que l'étape suivante consomme :

```
TypeScript project ──extract──▶ JSONL graph ──load──▶ Kùzu database ──▶ query / webview / report
                                (./.ts_knowledge_graph/graph/)            (./.ts_knowledge_graph/graph.kuzu)
                                                                    ▲
                                          V8 .cpuprofile ──enrich──┘
                                (enrich and cluster annotate the database in place)
```

### Construire le graphe

| Commande | Objectif |
| --- | --- |
| [`extract`](/commands/extract) | Analyse un projet TypeScript pour produire un graphe de connaissances JSONL. |
| [`load`](/commands/load) | Importe le graphe JSONL dans une base de données Kùzu embarquée. |

### Enrichir le graphe

| Commande | Objectif |
| --- | --- |
| [`enrich`](/commands/enrich) | Ingère un profil CPU V8 et attache des métriques d'exécution mesurées (`metadata.runtime`) aux nœuds. |
| [`cluster`](/commands/cluster) | Détecte les communautés de code avec l'algorithme de Leiden et attache l'indice de module (`metadata.community`) aux nœuds. |

### Interroger le graphe

| Commande | Objectif |
| --- | --- |
| [`find`](/commands/find) | Résout un nom (sous-chaîne) en identifiants de nœuds. Le point d'entrée de toutes les autres requêtes. |
| [`who-calls`](/commands/who-calls) | Appelants directs d'un symbole. |
| [`calls`](/commands/calls) | Ce qu'un symbole appelle directement. |
| [`references`](/commands/references) | Tout ce qui référence un symbole ou un type (appels, utilisation de type, héritage, instanciation, lectures). |
| [`neighbors`](/commands/neighbors) | Voisinage à un saut d'un nœud, entrant et sortant, tous types d'arêtes confondus. |
| [`blast-radius`](/commands/blast-radius) | Tout symbole impacté de manière transitive par la modification d'un nœud. |
| [`dead-exports`](/commands/dead-exports) | Symboles exportés sans aucune référence entrante. |
| [`hotspots`](/commands/hotspots) | Classe les nœuds selon leur levier d'optimisation — temps propre d'exécution, fan-in, nombre d'appels ou rayon d'impact. |
| [`cost`](/commands/cost) | Propage le coût propre en coût inclusif et classe les nœuds selon leur part du total ; ou décompose le coût d'un nœud en attribution appelé/appelant. |

### Vérifier et mesurer une modification

| Commande | Objectif |
| --- | --- |
| [`verify`](/commands/verify) | Exécute les scripts `typecheck` + `test` du projet comme une unique barrière conserver/annuler pour une modification ; se replie sur la seule vérification de types, signalée honnêtement, en l'absence de script de test. |
| [`benchmark`](/commands/benchmark) | Mesure la métrique d'exécution d'un nœud cible (profile → enrich → cost) sur N exécutions et rapporte la médiane + la dispersion, avec un delta indicatif baseline→after. |

### Utiliser le graphe

| Commande | Objectif |
| --- | --- |
| [`webview`](/commands/webview) | Sert le graphe dans une visualisation web interactive. |
| [`report`](/commands/report) | Écrit un `CODEBASE_BRIEF` — un instantané unique de la structure, de l'impact, de l'exécution et des frontières (markdown / JSON / PDF). |
| [`install`](/commands/install) | Installe les commandes slash de l'agent et la compétence `code-graph-query` dans le `.claude/` d'un projet. |

L'agent d'optimisation autonome n'est pas une commande de la CLI. Il s'agit de la
commande slash `/code-graph-optimize` de [Claude Code](https://claude.com/claude-code)
(voir [Agent](/agent/slash-commands)), qui appelle les commandes d'interrogation
ci-dessus pour trouver une modification vérifiée comme sûre et l'appliquer.

## Conventions communes

- **Les identifiants de nœuds proviennent toujours d'une requête.** Exécutez [`find`](/commands/find) (ajoutez `--json`)
  ou lisez un identifiant dans la sortie d'une autre requête. Les identifiants encodent la ligne de déclaration
  (`kind:relPath#name@line`) et changent dès que le code se déplace ; ne les écrivez donc jamais
  à la main et ne les réutilisez jamais d'une extraction à l'autre.
- **`-o, --output-folder <dir>`** vaut par défaut `./.ts_knowledge_graph` sur chaque commande (la
  base de données est lue depuis `<dir>/graph.kuzu`), conformément à la sortie par défaut de `load`.
  Modifiez-le pour interroger une base de données ailleurs.
- **`--json`** est disponible sur chaque commande d'interrogation et émet exactement la
  forme lisible par machine que l'agent d'optimisation consomme.
