# Install

`ts-knowledge-graph` is published to npm as
[`ts-knowledge-graph`](https://www.npmjs.com/package/ts-knowledge-graph). Every
command in these docs runs through `npx`, so there is **no global install** to
manage — you point the binary at any TypeScript project and go.

## Prerequisites

- **Node.js ≥ 20.12** (check with `node --version`).
- The target project must have a **`tsconfig.json`** — the extractor loads it
  through `ts-morph` from that config.
- **For the optimization agent only:** [Claude Code](https://claude.com/claude-code).
  The agent ships as the `/code-graph-optimize` slash command, so there is no API
  key or LLM provider to configure — the agent runtime is Claude Code itself.

## Run it on your own project

No install step. From the root of any project with a `tsconfig.json`:

```bash
npx ts-knowledge-graph extract . --semantic
npx ts-knowledge-graph load
npx ts-knowledge-graph dead-exports
```

`npx` fetches the published binary on first use and caches it. See
[The pipeline](/getting-started/pipeline) for the full walk-through.

## Clone the repository

To try the tool on its own source, or to add your own traversal, clone the repo
and install its dependencies:

```bash
git clone https://github.com/jeromeetienne/ts_knowledge_graph.git
cd ts_knowledge_graph
npm install
```

From a checkout you can run the CLI through the bundled npm scripts
(`npm run extract -- . --semantic`, `npm run dev -- load`, …) or, as the docs do,
with `npx ts-knowledge-graph <command>`.
